## 2. 필요한 프로그램 설치

### 2-1. Node.js 설치 (필수)

- JavaScript로 만든 프로그램을 실행하는 프로그램이에요

## 4. YouTube API 키 발급

### 4-1. Google Cloud Console 접속

**API 키가 왜 필요한가요?**
- YouTube 데이터를 가져오려면 구글의 허락이 필요해요
- API 키는 "나는 허락받았어요" 증명서 같은 거예요

**발급 방법:**

1. **Google Cloud Console 접속**
   - 주소: https://console.cloud.google.com
   - 구글 계정으로 로그인

2. **새 프로젝트 만들기**
   - 왼쪽 위 프로젝트 선택 → "새 프로젝트"
   - 프로젝트 이름: `YouTube-Consultant`
   - "만들기" 클릭

3. **YouTube Data API v3 활성화**
   - 왼쪽 메뉴 → "API 및 서비스" → "라이브러리"
   - 검색창에 "YouTube Data API v3" 입력
   - 첫 번째 결과 클릭
   - "사용 설정" 버튼 클릭

4. **API 키 만들기**
   - 왼쪽 메뉴 → "API 및 서비스" → "사용자 인증 정보"
   - 상단 "+ 사용자 인증 정보 만들기" → "API 키" 선택
   - API 키가 생성됩니다!

5. **API 키 복사**
   - 생성된 키를 복사해서 메모장에 저장하세요
   - 예시: `AIzaSyB1234567890abcdefghijklmnopqrstuv`

### 4-2. API 키 사용 제한 설정 (선택, 하지만 추천)

**보안을 위해 설정하세요:**

1. 생성된 API 키 옆 "제한사항 수정" 클릭

2. **애플리케이션 제한사항**
   - "HTTP 리퍼러(웹사이트)" 선택
   - 배포 후 사이트 주소 입력 (예: `https://your-app.vercel.app/*`)

3. **API 제한사항**
   - "키 제한" 선택
   - "YouTube Data API v3"만 체크

4. "저장" 클릭

## 6. 배포하기

### 배포가 뭔가요?

- 지금까지는 내 컴퓨터에서만 볼 수 있었어요
- 배포하면 인터넷에 올라가서 친구들도 볼 수 있어요!
- 무료 배포 서비스를 사용할 거예요

### 6-1. Vercel로 배포 (추천, 가장 쉬움)

**Vercel이 뭔가요?**
- 웹사이트를 무료로 인터넷에 올려주는 서비스예요
- GitHub과 연동해서 자동으로 배포해줘요

#### STEP 1: Vercel 계정 만들기

1. **Vercel 홈페이지 접속**
   - 주소: https://vercel.com

2. **회원가입**
   - "Sign Up" 또는 "시작하기" 클릭
   - "Continue with GitHub" 선택
   - GitHub 계정으로 로그인
   - 권한 승인

#### STEP 2: GitHub에 코드 올리기

**GitHub이 필요한 이유:**
- Vercel이 GitHub에서 코드를 가져와서 배포해요
- 코드를 수정하면 자동으로 다시 배포돼요

1. **GitHub 홈페이지 접속**
   - 주소: https://github.com
   - 로그인

2. **새 저장소 만들기**
   - 오른쪽 위 "+" 버튼 → "New repository"
   - Repository name: `youtube-consultant`
   - Public/Private 선택 (Public 추천)
   - "Create repository" 클릭

3. **로컬 코드를 GitHub에 올리기**
   - 터미널에서 프로젝트 폴더로 이동
   
   ```bash
   # Git 초기화 (처음 한 번만)
   git init
   
   # 모든 파일 추가
   git add .
   
   # 커밋 (변경사항 저장)
   git commit -m "첫 배포 준비"
   
   # GitHub 연결
   git remote add origin https://github.com/내아이디/youtube-consultant.git
   
   # 업로드
   git branch -M main
   git push -u origin main
   ```

4. **GitHub에서 확인**
   - GitHub 저장소 새로고침
   - 파일들이 올라왔으면 성공!

#### STEP 3: Vercel에서 배포

1. **Vercel 대시보드**
   - https://vercel.com/dashboard
   - "Add New..." → "Project" 클릭

2. **GitHub 저장소 연결**
   - "Import Git Repository" 섹션
   - GitHub 계정 연결 (처음 한 번만)
   - `youtube-consultant` 저장소 찾기
   - "Import" 클릭

3. **프로젝트 설정**
   - Project Name: `youtube-consultant` (그대로 두기)
   - Framework Preset: Vite (자동 감지됨)
   - Root Directory: `./` (그대로 두기)
   - Build Command: `npm run build` (자동)
   - Output Directory: `dist` (자동)

4. **환경 변수 설정 (선택)**
   - "Environment Variables" 섹션
   - 필요하면 API 키 등 추가
   - 우리 프로젝트는 클라이언트에서 API 키 입력하므로 불필요

5. **배포 시작**
   - "Deploy" 버튼 클릭!
   - 2-3분 기다리기

6. **배포 완료**
   ✅ 성공 화면이 나옵니다!
   ✅ 주소가 생성됩니다: `https://youtube-consultant-abc123.vercel.app`

7. **사이트 접속**
   - 생성된 주소 클릭
   - 내 사이트가 인터넷에 올라갔어요! 🎉

#### STEP 4: 커스텀 도메인 연결 (선택)

**내 도메인이 있다면:**

1. Vercel 프로젝트 대시보드 → "Settings" → "Domains"
2. 도메인 입력 (예: `mysite.com`)
3. DNS 설정 안내에 따라 설정
4. 24시간 이내 연결 완료

### 6-3. GitHub Pages로 배포 (무료, 하지만 복잡함)

**GitHub Pages는 정적 사이트 호스팅이에요**

1. **package.json에 homepage 추가**
   ```json
   {
     "homepage": "https://내아이디.github.io/youtube-consultant"
   }
   ```

2. **gh-pages 패키지 설치**
   ```bash
   npm install --save-dev gh-pages
   ```

3. **package.json에 스크립트 추가**
   ```json
   {
     "scripts": {
       "predeploy": "npm run build",
       "deploy": "gh-pages -d dist"
     }
   }
   ```

4. **배포**
   ```bash
   npm run deploy
   ```

5. **GitHub 설정**
   - GitHub 저장소 → Settings → Pages
   - Source: `gh-pages` 브랜치 선택
   - Save

## 7. 문제 해결

### 자주 발생하는 문제들

#### 문제 1: `npm install` 실행 시 에러

**증상:**
```
npm ERR! code ENOENT
```

**해결 방법:**
1. `package.json` 파일이 있는 폴더인지 확인
2. `ls` (맥) 또는 `dir` (윈도우) 명령어로 파일 확인
3. 올바른 폴더로 이동: `cd [프로젝트 폴더]`

#### 문제 2: `npm run dev` 실행 시 포트 에러

**증상:**
```
Port 5173 is already in use
```

**해결 방법:**
1. 이미 다른 프로그램이 5173 포트를 쓰고 있어요
2. 방법 1: 다른 프로그램 종료
3. 방법 2: 다른 포트 사용
   ```bash
   npm run dev -- --port 3000
   ```

#### 문제 3: YouTube API 키 에러

**증상:**
```
API key not valid
```

**해결 방법:**
1. API 키 올바르게 복사했는지 확인 (공백 없이)
2. YouTube Data API v3 활성화 확인
3. API 키 생성 후 5분 정도 기다리기
4. 브라우저 캐시 삭제 후 재시도

#### 문제 4: 배포 후 API 키 제한 에러

**증상:**
```
This IP, site or mobile application is not authorized
```

**해결 방법:**
1. Google Cloud Console → API 키 설정
2. "HTTP 리퍼러" 설정에 배포된 주소 추가
   - 예: `https://your-app.vercel.app/*`
3. 5-10분 후 다시 시도

#### 문제 5: 빌드 에러

**증상:**
```
Build failed
```

**해결 방법:**
1. 로컬에서 빌드 테스트
   ```bash
   npm run build
   ```
2. 에러 메시지 확인 및 수정
3. 수정 후 GitHub에 푸시
   ```bash
   git add .
   git commit -m "빌드 에러 수정"
   git push
   ```

#### 문제 6: Node.js 버전 에러

**증상:**
```
Error: The engine "node" is incompatible
```

**해결 방법:**
1. Node.js 버전 확인
   ```bash
   node --version
   ```
2. Node.js 18 이상 필요
3. 낮은 버전이면 최신 LTS 버전 재설치

## 8. 추가 팁

### 8-1. 코드 수정 후 재배포

**GitHub + Vercel/Netlify를 사용하면:**

1. 코드 수정
2. GitHub에 푸시
   ```bash
   git add .
   git commit -m "수정 내용 설명"
   git push
   ```
3. 자동으로 재배포됩니다! (2-3분 소요)

### 8-2. 로그 확인

**배포 중 문제가 생기면:**

- Vercel: 프로젝트 → Deployments → 최신 배포 클릭 → Logs
- Netlify: Site → Deploys → 최신 배포 클릭 → Deploy log

### 8-3. 배포 취소 (롤백)

**잘못 배포했다면:**

- Vercel/Netlify 대시보드에서 이전 버전으로 롤백 가능
- 또는 GitHub에서 이전 커밋으로 돌아가기

### 8-4. 모니터링

**사이트가 잘 작동하는지 확인:**

- Vercel/Netlify에서 자동 모니터링 제공
- 다운타임 발생 시 이메일 알림

## 10. 용어 설명

**초보자를 위한 개발 용어 해설:**

- **Node.js**: JavaScript를 실행하는 프로그램
- **npm**: Node.js 패키지(부품)을 설치하는 도구
- **Git**: 코드를 저장하고 관리하는 도구
- **GitHub**: Git으로 관리하는 코드를 인터넷에 올리는 곳
- **API**: 다른 서비스의 데이터를 가져오는 통로
- **API 키**: API를 쓸 수 있는 열쇠
- **배포**: 인터넷에 웹사이트를 올리는 것
- **로컬**: 내 컴퓨터
- **서버**: 인터넷에 있는 컴퓨터
- **포트**: 프로그램이 쓰는 문 (번호)
- **빌드**: 코드를 웹사이트로 만드는 과정
- **커밋**: Git에 변경사항 저장
- **푸시**: GitHub에 업로드
- **클론**: GitHub에서 다운로드
